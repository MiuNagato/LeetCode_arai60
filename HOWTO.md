# LeetCode 取り組み方メモ

## 基本フロー

1. 問題を読む（5分）
2. 解法を考える（15〜20分）
3. 解けなければヒントを見る、それでもダメなら解説を見る
4. コードを書く
5. テストケースを手動で追う
6. 提出して結果を確認
7. 他の解法をリサーチして学ぶ

## 解法メモのテンプレート

各問題フォルダに `note.md` を作る。

```markdown
## 問題番号 - 問題名

### 難易度
Easy / Medium / Hard

### 解法
- アプローチ名（例: ハッシュマップ、二分探索）

### 計算量
- 時間: O(n)
- 空間: O(n)

### ポイント
- 気づいたこと、ハマったポイントを書く
```

## よく使うパターン

| パターン | 使いどころ |
|---------|-----------|
| スライディングウィンドウ | 連続する部分列・部分配列 |
| 二分探索 | ソート済み配列、単調な条件 |
| BFS | 最短経路、レベル順探索 |
| DFS | 全探索、バックトラック |
| DP | 最適化問題、重複部分問題 |
| ハッシュマップ | ペア探索、頻度カウント |
| 二分木 | 再帰で左右に分けて考える |

## 心がけ

- 最初はBrute Forceから考える
- 計算量を意識して改善できないか考える
- 同じ問題を時間を置いて解き直す
